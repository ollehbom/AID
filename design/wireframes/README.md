# Wireframes

This directory contains structured wireframe JSONs generated by the Design Agent.

## Purpose

Wireframes provide a machine-readable, structured representation of UI layouts that can be:

- Translated directly into code by the Dev Agent
- Version controlled alongside design specs
- Easily compared and diff'd
- Used to generate visual previews

## Format

Each wireframe follows a nested structure based on Figma's design paradigm:

```json
{
  "name": "Screen Name",
  "type": "FRAME",
  "width": 375,
  "height": 812,
  "children": [
    {
      "name": "Component Name",
      "type": "FRAME|TEXT|RECTANGLE|etc",
      "properties": "...",
      "children": []
    }
  ]
}
```

## Node Types

- **FRAME**: Container for other elements (like a div or view)
- **TEXT**: Text content with styling
- **RECTANGLE**: Solid shapes, backgrounds, dividers
- **IMAGE**: Image placeholders
- **BUTTON**: Interactive button elements
- **INPUT**: Form input fields

## Properties

Common properties across node types:

- `name`: Semantic name (e.g., "HeaderText", "CTAButton")
- `width`/`height`: Fixed pixel values or "FILL"/"HUG"
- `layoutSizingHorizontal`/`layoutSizingVertical`: "FILL", "HUG", or "FIXED"
- `backgroundColor`: Hex color code
- `cornerRadius`: Border radius in pixels
- `fontSize`: Font size for text
- `characters`: Text content
- `padding`: Spacing inside elements
- `gap`: Spacing between children

## Usage

These wireframes are consumed by the Dev Agent to:

1. Generate initial component structure
2. Apply semantic naming conventions
3. Implement responsive layouts
4. Ensure design-dev consistency
