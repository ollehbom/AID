## Foundational UI Design Specification

### 1. User Flow: Initial Login & Dashboard Access

1.  **Access Application**: User opens the web application URL.
2.  **Google Login Screen Display**: The application presents a dedicated login screen.
3.  **Initiate Login**: User clicks the "Sign in with Google" button.
4.  **Google Authentication**: User is redirected to Google's authentication flow.
5.  **Redirect & Authenticate**: Upon successful authentication with Google, the user is redirected back to the application.
6.  **Dashboard Load**: The application loads the main dashboard content.
7.  **Dashboard Display**: The dashboard displays mock content, indicating successful access.

### 2. All Possible States

#### 2.1. Google Login Screen
*   **Initial State**: Displays "Welcome!" header, "Sign in with Google" button.
*   **Loading State (during redirect)**: Button might show a spinner or be disabled.
*   **Error State (Google Auth Failure)**: Displays an error message below the button.

#### 2.2. Dashboard Screen
*   **Loading State**: Displays a full-screen or central spinner while dashboard content is fetched.
*   **Success State**: Displays the main dashboard content (e.g., "Your Dashboard", content cards).
*   **Empty State**: If no content is available after loading, displays a message like "No data to display yet." (Future state, not MVP).
*   **Error State (Content Fetch Failure)**: Displays an error message and a retry option.

### 3. Exact Copy/Microcopy

#### 3.1. Google Login Screen
*   **Header**: "Welcome!"
*   **Button**: "Sign in with Google"
*   **Error Message (Auth Failure)**: "Failed to sign in with Google. Please try again or contact support."
*   **Support Link (optional)**: "Contact support"

#### 3.2. Dashboard Screen
*   **Header**: "Your Dashboard"
*   **Welcome Message**: "Welcome back!"
*   **Empty State Message**: "No data to display yet. Start by [Action suggestion, e.g., 'creating a new project']." (Future state, not MVP).
*   **Error Message (Content Failure)**: "Failed to load dashboard content. Please try again."
*   **Retry Button**: "Refresh"

### 4. Error Handling and Edge Cases

*   **Google Authentication Failure**:
    *   **Handling**: Display a clear, concise error message on the login screen.
    *   **Edge Case**: User closes Google auth window.
    *   **Handling**: Return to the login screen without an error, allowing the user to retry.
*   **Dashboard Content Loading Failure**:
    *   **Handling**: Display an error message within the dashboard area, with a prominent "Refresh" button.
    *   **Edge Case**: Network outage during content fetch.
    *   **Handling**: Display network error message, suggest checking connection.
*   **Unauthorized Access**:
    *   **Handling**: If a user somehow bypasses login or their session expires, redirect them to the Google Login screen.

### 5. Transitions and Animations

*   **Login to Dashboard**: A subtle page transition or fade-in effect upon successful authentication.
*   **Loading States**: Use standard shadcn/ui spinner components for content loading.
*   **Button Interactions**: Standard hover/active states for buttons.
