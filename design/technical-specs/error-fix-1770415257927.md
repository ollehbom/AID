# Technical Specification: Google Login & Basic Dashboard Frontend

## Architecture Overview
The frontend application will be a Single-Page Application (SPA) built with React, hosted as static assets on a CDN (e.g., AWS S3 + CloudFront, Vercel, Netlify). It will interact with a separate backend API for secure Google OAuth token verification and session management. The application will feature a clear separation between public (Login) and protected (Dashboard) routes.

## Components

- **React Application Core**: The main application scaffold, managing global state, routing, and component orchestration.
- **Shadcn UI**: The primary component library, providing pre-built, customizable, and accessible UI elements (e.g., Button, Input, Card).
- **Google OAuth Client Integration**: A client-side library (e.g., `@react-oauth/google`) to initiate and handle the Google authentication flow, obtaining the ID token.
