# .env Setup Quick Reference

## First Time Setup

```bash
# 1. Copy the example file
cp .env.example .env

# 2. Edit the file (use any text editor)
nano .env
# or
code .env
# or
vim .env

# 3. Replace the placeholder with your actual key
# Change: OPENAI_API_KEY=sk-your-api-key-here
# To:     OPENAI_API_KEY=sk-proj-abc123xyz...

# 4. Save and close

# 5. Verify it's working
python scripts/invoke_product_agent.py test-setup
```

## What's in .env

```bash
# Required
OPENAI_API_KEY=sk-proj-abc123...

# Optional overrides
MODEL=gpt-4.1          # Change model (gpt-4o, gpt-4-turbo, etc.)
TEMPERATURE=0.7        # Adjust creativity (0.0-2.0)
```

## Security Checklist

- [x] `.env.example` committed (template only, no real keys)
- [x] `.env` in `.gitignore` (never committed)
- [x] `.env` added to local directory (with real key)
- [ ] GitHub Actions secret configured (for CI/CD)

## Verify Your Setup

```bash
# Check .env exists
ls -la .env

# Check it has your key
cat .env | grep OPENAI_API_KEY

# Check it's NOT tracked by git
git status | grep .env
# Should NOT appear - this is good!

# Test it works
bash scripts/test_product_agent.sh
```

## Troubleshooting

### "OPENAI_API_KEY not found"

```bash
# Create .env from example
cp .env.example .env

# Edit and add your key
nano .env
```

### "Invalid API key"

```bash
# Verify your key at:
# https://platform.openai.com/api-keys

# Make sure it starts with 'sk-'
# Make sure there are no spaces or quotes around it
```

### ".env is showing in git status"

```bash
# Check .gitignore
cat .gitignore | grep .env

# If it's there but still showing, remove from tracking:
git rm --cached .env

# Then verify:
git status | grep .env
# Should not appear
```

## For Different Environments

### Local Development

```bash
# Just use .env
cp .env.example .env
# Add your dev API key
```

### GitHub Actions

```bash
# Repository Settings → Secrets → Actions
# Add: OPENAI_API_KEY
# Value: Your API key
```

### Team Setup

```bash
# Each team member:
cp .env.example .env
# Each person adds their own key

# Never share .env files
# Never commit .env to git
```

## Benefits Over Export

| Method                      | Persistent         | Secure              | Team-Friendly             |
| --------------------------- | ------------------ | ------------------- | ------------------------- |
| `export OPENAI_API_KEY=...` | ❌ Lost on restart | ❌ In shell history | ❌ Hard to share setup    |
| `.env` file                 | ✅ Saved locally   | ✅ Git ignored      | ✅ Example file shareable |

## What Gets Committed vs Not

✅ **Committed to Git:**

- `.env.example` - Template
- `.gitignore` - Protection
- `SECURITY.md` - Best practices

❌ **NEVER Committed:**

- `.env` - Your actual key
- `.env.local` - Local overrides
- Any file with real API keys
